# Project Status Skill

## Overview

プロジェクトの現状を自動的に分析し、次に取り組むべきタスクを提案するスキル。

## Usage

```
/project-status
```

## Behavior

1. **GitHub MCP で情報取得**
   - Open Issues を取得（ラベル、マイルストーン付き）
   - Open PRs を取得（レビュー状態、CIステータス）
   - 最近クローズしたIssues（直近5件）

2. **Git 状態確認**
   - 現在のブランチ
   - 未コミット変更の有無
   - リモートとの差分

3. **優先度分析**
   - ラベル（`priority: critical/high/medium/low`）
   - 依存関係（本文中の `Depends on #X`）
   - マイルストーン期限
   - ブランチの存在（進行中タスク）

4. **次のタスク提案**
   - 最優先タスク（1-3件）
   - 理由（優先度、依存関係、ブロック状況）

5. **MEMORY.md 更新**
   - 現在の状況を記録
   - Open Issues リスト
   - 最近の完了タスク

## Output Format

```
【プロジェクト状況】
📍 現在のブランチ: develop
📊 Open Issues: 5件（高: 2, 中: 2, 低: 1）
🔧 Open PRs: 1件（レビュー待ち）

【次のタスク（推奨順）】
1. Issue #65: デザインシステム刷新
   - 優先度: 高
   - 理由: 他の3件のIssueをブロック中

2. Issue #66: パフォーマンス最適化
   - 優先度: 中
   - 理由: マイルストーン「Phase 1」（期限: 2026-02-15）

【最近の完了タスク】
✅ #64: セキュリティヘッダー追加（2026-02-07）
✅ #63: 言語切り替え不具合修正（2026-02-06）

どのIssueに着手しますか？
```

## Notes

- GitHub MCP のみ使用（gh CLI 不使用）
- MEMORY.md は自動更新（手動編集不要）
- 優先度が同じ場合は Issue番号が小さい順
